## About This Session

Azure Storage is where most cloud applications keep their files, blobs, tables, and queues. Securing it correctly is critical — a misconfigured storage account is one of the most common causes of cloud data breaches.

This session covers how to lock down storage accounts: disabling public access, using Shared Access Signatures (SAS tokens) to grant time-limited access, enabling encryption at rest and in transit, and configuring Azure Defender for Storage to detect unusual access patterns.

**By the end of this session you will be able to:**
- Identify common Azure Storage security misconfigurations
- Configure access keys, SAS tokens, and RBAC for storage accounts
- Enable encryption and enforce HTTPS-only access
- Set up Microsoft Defender for Storage alerts
