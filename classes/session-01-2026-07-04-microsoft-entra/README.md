## About This Session

Microsoft Entra is Microsoft's cloud identity platform — it controls who can sign in to your apps and what they are allowed to do once they are in.

This session covers how to protect access to cloud resources using Entra ID. You will learn about Conditional Access (rules that check things like location and device before allowing sign-in), Multi-Factor Authentication (requiring a second proof of identity), Privileged Identity Management (limiting admin access to only when it is needed), and how roles are assigned to users and groups.

**By the end of this session you will be able to:**
- Explain what Microsoft Entra ID does and why it is central to the SC-500 exam
- Configure Conditional Access policies to restrict risky sign-ins
- Understand how MFA reduces the risk of stolen passwords
- Describe how Privileged Identity Management (PIM) limits standing admin access
